Danger_RSA·

题目：Danger_RSA的附件.zip

这个是2023 Crypto CTF原题，可以直接套春乎或V的公众号的WP解，但其实这里并不需要爆破eee的因子恢复sss和ttt，下面细说。

首先根据题目有p=Xpa+sp = X_p^a + sp=Xpa​+s、q=Xqa+tq = X_q^a + tq=Xqa​+t和e=ste = ste=st，乘一下有

tXqa+sXpa=N−e−(XpXq)at X_q^a + s X_p^a = N - e - (X_p X_q)^a tXqa​+sXpa​=N−e−(Xp​Xq​)a

Easy_3L·

题目：Easy_3L的附件.zip

题目套了几层，首先是一个种子为mmm的线性同余LCG，给出S1、S2和S4、S5，由于不知道模数，只根据这几个无法求出LCG的系数aaa和bbb，还需要S3。S3被一个一维的NTRU加密，LLL老套路规约一下可以求密钥然后解密（只是变量名被换了）。所以这就是个LCG+LLL的模板题。

首先解NTRU的密钥，根据密钥生成有h≡f−1q(modp)h \equiv f^{-1}q \pmod ph≡f−1q(modp)，即fh−kp=qfh - kp = qfh−kp=q，令

v=(f,−k)B=[1Dh0Dp]w=(f,Dq)v = (f, -k) \\ B = \begin{bmatrix} 1 & D h \\ 0 & Dp \end{bmatrix} \\ w = (f, D q) v=(f,−k)B=[10​DhDp​]w=(f,Dq)

可以构造出线性方程vB=wvB = wvB=w，对BBB进行LLL规约得fff和qqq。

PS：其中DDD是配平数，这个DDD可以控制上面向量www的大小，从而可以控制规约出来的qqq的大小。这里说明一下，只要对BBB规约成功就可以解出一组合法的(f,q)(f, q)(f,q)，即符合h≡f−1q(modp)h \equiv f^{-1}q \pmod ph≡f−1q(modp)，但是如果需要解密成功的话需要足够大的qqq，否则会因为模qqq操作造成S3S_3S3​值得损失。为了让qqq足够大，这里的DDD需要足够小，但是不是每个DDD都可以规约成功，所以后面exp中对DDD从小到大进行枚举。

有了密钥后就是NTRU解密出S3S_3S3​，

S3=(cf(modp))∗f−1mod  qS_3 = (c f \pmod p) * f^{-1} \mod q S3​=(cf(modp))∗f−1modq

再然后根据题目的LCG有

Si≡aSi−1+b(modn)S_{i} \equiv a S_{i-1} + b \pmod n Si​≡aSi−1​+b(modn)

先凑够两个相减消去碍事的bbb

Si+1−Si≡a(Si−Si−1)(modn)S_{i+1} - S_i \equiv a (S_i - S_{i-1}) \pmod n Si+1​−Si​≡a(Si​−Si−1​)(modn)

然后提取出aaa

(Si+1−Si)(Si−Si−1)−1≡a(modn)(S_{i+1} - S_i) (S_i - S_{i-1})^{-1} \equiv a \pmod n (Si+1​−Si​)(Si​−Si−1​)−1≡a(modn)

但是现在并不知道模数nnn，再凑个jjj就有

(Sj+1−Sj)(Sj−Sj−1)−1≡a(modn)(S_{j+1} - S_j) (S_j - S_{j-1})^{-1} \equiv a \pmod n (Sj+1​−Sj​)(Sj​−Sj−1​)−1≡a(modn)

所以

(Si+1−Si)(Si−Si−1)−1≡(Sj+1−Sj)(Sj−Sj−1)−1(modn)(Si+1−Si)(Sj−Sj−1)−(Sj+1−Sj)(Si−Si−1)≡0(modn)(S_{i+1} - S_i) (S_i - S_{i-1})^{-1} \equiv (S_{j+1} - S_j) (S_j - S_{j-1})^{-1} \pmod n \\ (S_{i+1} - S_i)(S_j - S_{j-1}) - (S_{j+1} - S_j)(S_i - S_{i-1}) \equiv 0 \pmod n (Si+1​−Si​)(Si​−Si−1​)−1≡(Sj+1​−Sj​)(Sj​−Sj−1​)−1(modn)(Si+1​−Si​)(Sj​−Sj−1​)−(Sj+1​−Sj​)(Si​−Si−1​)≡0(modn)

(Si+1−Si)(Sj−Sj−1)−(Sj+1−Sj)(Si−Si−1)=kijn(S_{i+1} - S_i)(S_j - S_{j-1}) - (S_{j+1} - S_j)(S_i - S_{i-1}) = k_{ij} n (Si+1​−Si​)(Sj​−Sj−1​)−(Sj+1​−Sj​)(Si​−Si−1​)=kij​n

SigninCrypto·

题目：SigninCrypto的附件.zip

flag被DES3加密，首先分析IV，可得digest1和digest2相等，所以IV1和IV2相等。

分析hint2，解码一下就可以看到前4 byts是可见字符GWHT，算一下长度，发现hint2只是抹去了IV得低4 bytes，所以可知IV1和IV2为4 bytes长，且都为GWHT，于是恢复了IV。

然后KEY = K1 + K2 + K3，首先分析长度，DES3的密钥长度是16或24（否则会报VallueError），K2和K3都是8，所以KEY应该是24，即K1长度是8，再分析xor，hint实际有效得只有2 bytes，即os.urandom(2)，而hint1长度为16 bytes，即hint1的前8 bytes由4个os.urandom(2)重复，估可恢复hint1，异或回去得K1。

Rand()的循环长度明显提示了随机数预测，由于每次循环前都重置一样的seed，根据MT19937可由List1恢复List2，即List1中刚好含有List2输出时被抹去的信息，然后就有624个32比特，上RandCrack预测随机数。

flag头是DASCTF，爆破1 bytes得K3。

最后解密得flag。
